# Практична робота №6

## Завдання варіант №8

Побудувати приклад невизначеної поведінки, який демонструє різну роботу при компіляції з -O0 і -O2, та пояснити вплив оптимізацій компілятора.

### Ідея реалізації

У програмі змінна `x` оголошена, але не ініціалізована. Читання такої змінної є невизначеною поведінкою `undefined behavior`, тому стандарт C не гарантує конкретного результату.
При компіляції з `-O0` оптимізація майже не застосовується, і програма читає випадкове значення `x` зі стеку, після чого перевіряє умову `if (x == 0)`.
При компіляції з `-O2` компілятор застосовує оптимізації і може змінити або спростити код, припускаючи, що програма не містить `undefined behavior`. У результаті програма може завжди виводити лише один варіант `zero` або `non-zero`, навіть якщо фактичне значення змінної невідоме.
Отже, різниця пояснюється тим, що при `-O0` програма працює з випадковим значенням зі стеку, а при `-O2` оптимізатор може змінити поведінку коду через наявність невизначеної поведінки

### Збірка та запуск 

```bash
gcc task5_8.c -O0 -o task5_8_O0
./task5_8_O0

gcc task5_8.c -O2 -o task5_8_O2
./task5_8_O2
```

### Висновки

Невизначена поведінка в C може призводити до різних результатів виконання програми. Рівень оптимізації компілятора (`-O0`, `-O2`) може змінювати поведінку коду, якщо в ньому є помилки, такі як використання неініціалізованих змінних. Тому змінні потрібно завжди ініціалізувати.
